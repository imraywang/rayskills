# 长文原型

只选一个主原型。原型决定文章怎样推进，不决定作者必须使用某种固定语气。

## 现象解读

适合从公开观点、行业变化或平台动作推出更大的判断。

推进方式：具体变化 → 常见解释 → 信号为何失效 → 核心迁移 → 最强反方 → 对平台或个人的影响 → 余波。

重点是让读者重新理解一件已经看见的事。通常使用连续短段落，不必设置小标题。

## 观点论证

适合作者已经有明确主张，需要证明它为何成立。

推进方式：争议命题 → 证据和机制 → 边界 → 反例 → 修正后的更强判断。

不要只找支持材料。最强反方必须让原判断发生收缩、补充或升级。

## 个人实践

适合真实工具体验、工作流搭建、项目复盘和做事过程。

推进方式：真实目标或痛点 → 最初做法 → 失败或返工 → 关键改变 → 可验证结果 → 得出的判断 → 他人如何最小化复用。

事实价值高于戏剧性。不能补造失败、对话、耗时和效果。为了可读性可以调整叙述顺序，但不能改变事实因果。

## 案例拆解

适合公司、产品、人物或一线实践案例。

推进方式：结果或反常识 → 背景 → 关键选择 → 运行机制 → 为什么有效 → 失效边界 → 可迁移判断。

不要把单个案例写成普遍规律。明确案例代表什么，不代表什么。

## 教程与方法

适合读者需要照着完成任务。

推进方式：适用对象和结果 → 前置条件 → 判断原则 → 分步操作 → 常见失败 → 验收方式 → 下一步。

教程必须使用清楚的小标题、步骤和必要清单。不要把现象解读型“不设标题”的规则套进来。先讲为什么这样判断，再给操作。

## 选择规则

- 材料的价值主要来自公开信号及其含义：现象解读。
- 价值主要来自明确立场和争议：观点论证。
- 价值主要来自用户亲自做过：个人实践。
- 价值主要来自一个外部对象的运行方式：案例拆解。
- 价值主要来自读者要完成一件事：教程与方法。

如果两种都像，问文章删去哪一部分就不成立。不可删除的那部分对应主原型。
