# 公众号草稿恢复手册

| 症状 | 原因 | 恢复动作 |
|---|---|---|
| 草稿箱出现同题多份 | 未读取本地 media ID 或先新建后查重 | 停止创建；使用 frontmatter 的 ID 更新原草稿 |
| 中文变成 `Ã¥...` 或 `å...` | HTTP 响应被错误解码后又写回 | 用本地正确 HTML 覆盖正文；标题取 frontmatter；摘要取明确文案；强制 UTF-8 回读 |
| 标题正确但正文乱码 | 只核对 API 成功码，没有检查正文 | 回读首句、末句和全部原文块；不通过就更新同一草稿 |
| 封面丢失 | 更新请求没有保留 `thumb_media_id` | 先 `draft/get` 读取原封面 ID，再更新原草稿 |
| 排版有大段空白 | Markdown 连续空行或 HTML 空 `<p>` | 回到本地清理并重跑 `prepare_article.py` |
| 全文太单调 | 只有普通段落和统一字号 | 增加章节、三项导读、少量金句和关键判断标记，不堆色块 |
| 英文词被下划线截断 | 自动选重点按字符窗口切分 | 重点只取完整中文短语或完整英文 token，重新生成 |
| 文末出现作者占位或三连 | 排版器自动附加签名组件 | 按用户署名策略整块删除，重新生成预览和正文 |
| API 报 IP 白名单或权限错误 | 公众号后台未授权当前出口或账号缺权限 | 保留本地交付包，不新建替代草稿；请用户处理后台权限后原地重试 |
| 更新后验证数量不一致 | 微信清洗了不兼容标签或遗漏原文 | 比对原文块、章节、空段落与重点数量；修正 HTML 后更新同一 ID |

## 乱码修复完成标准

不能只把标题修回来。必须同时确认：

1. 标题和摘要是正确中文。
2. 正文首句、末句和所有原文块按顺序存在。
3. 封面 ID 与修复前一致。
4. 章节、重点样式和空段落数量符合交付清单。
5. 署名策略仍然生效。

## 失败时如何汇报

说明当前停在哪一层：本地排版、授权、接口写入或回读验收。不要把“请求已发送”描述成“草稿已完成”，也不要把需要用户处理的白名单问题说成内容错误。
