# 公众号长文排版契约

## 目标

排版首先解决移动端分神，不用装饰堆叠掩盖文章结构。视觉层级服务三种动作：让人停下、让人扫读、让人继续读。

## 三层结构

| 层级 | 用途 | 建议频率 |
|---|---|---|
| 锚点 | 章节标题、核心判断、结论 | 全文 3–5 处 |
| 标记 | 原文已经确认的完整关键判断 | 按母稿语义使用 |
| 容器 | 导读、章节标题、引用、数据或提示 | 按文章结构使用 |

不要把每句话都做成锚点。不要用空段落、连续 `<br>` 或大面积 padding 模拟章节。

## 长文最低结构

- 3000–8000 字通常使用 4–9 个章节。
- 第一屏需要一个明确判断或冲突，但不要重复公众号外标题。
- 三个以上章节可以提炼三项导读；导读不是完整目录。
- 章节标题必须推进判断，避免“背景”“分析”“总结”。
- 原文中的完整关键判断保留加粗；排版器新增下划线时不得拆开英文单词。
- 默认不生成金句卡；只有用户明确需要更强视觉锚点时，最多选择 1–2 处。
- 结尾按用户要求决定：只保留文章结论、加 END、保留原文署名，或完全无尾注。不得自动加入作者占位、二维码和互动引导。

## HTML 交付约束

- 干净正文是单个顶层 `<section>` 片段，不含 `html`、`head`、`body`、`script`。
- 样式使用内联 CSS；不依赖外部字体、样式表或运行时脚本。
- 不使用空 `<p>`、连续 `<br>`、`position:fixed`、`display:grid` 或横向溢出。
- 正文图片自适应且不强制放大小图；封面不嵌入正文首屏。
- 预览页可以有复制按钮，但复制区域只包含干净正文片段。

## 主题选择

- 观点、力量感、争议判断和行动号召：`ray-judgment`（Ray 判断）。
- 教程、步骤、工具盘点和行动指南：`ray-method`（Ray 方法）。
- 科技评论、专业判断和长篇解释：`ray-deepread`（Ray 深读）。

三套主题的完整规则见 [theme-library.md](theme-library.md)。先选阅读动作，再选主题。用户只说“排版单调”时，优先检查章节、段落长度、加粗判断和留白，不增加更多卡片与颜色。
