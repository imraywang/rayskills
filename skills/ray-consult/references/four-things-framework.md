## 0. 约束三角(先框可行域)

方案设计第一步不是画架构,是框可行域。三条边,方案里每个组件都要能说出它压在三角形哪个位置之内;超出任何一条边的设计就是埋雷。

| 约束边 | 来源(诊断维度) | 决定什么 |
|---|---|---|
| **预算韧性** | S3 分数 + 需求考古的失败成本 | 不是"有多少钱",是"价值证明前能撑多久 + 见效后能追加多少"。决定分期粒度与 Phase 0 预算上限 |
| **组织承接力** | D5(IT) / O5(变革容量) / G5(运营资源) | 决定技术形态上限:SaaS vs 私有化、能养几个专职角色、变革节奏 |
| **数据现状** | K 维 + A1/A2 | 决定起点:从治理还债开始,还是可直接建消费层 |

反模式:"预算先做了再说""IT 那边让他们想办法"——把约束边当空气,方案在执行层被静默绞杀。

## 1. 建什么:参考架构四层

所有知识库方案共享同一张参考架构。方案 = 决定每层在这家企业用什么实现、哪几层在哪一期建。

- **L4 消费层**:人的界面(门户 / 搜索 / IM 内嵌入口 / 培训课程)+ AI 的界面(问答机器人 / Copilot / Agent,RAG 及以上)
- **L3 存储与检索层**:知识库平台(文档型 / 结构化型)、全文检索、向量检索
- **L2 治理层(方法论重心)**:SSOT 机制 + 采集清洗管道 + 结构化与元数据规范 + 审核流(含合规) + 代谢机制(复审 / 过期 / 下架) + 权限与密级
- **L1 源系统层**:业务系统(ERP/CRM/OMS/PIM)、文档存量、IM 沉淀、人脑隐性知识(经萃取进 L2)

### 三条架构纪律

1. **L2 是人与 AI 的共用地基**。预算被挤压到把 L2 砍薄的方案打回重做——省掉治理层等于把错误规模化。
2. **L4 的 AI 界面只消费经过 L2 的白名单语料**。严禁把共享盘直接挂 RAG——那是"十年没打扫的仓库直接开门营业"。
3. **L1 隐性知识要显式萃取机制**(专家访谈 / 师徒结对文档化 / 工单会话挖掘)。买平台不会让老师傅脑子里的东西自动显性化。

### 知识单元设计(L2 第一个工程动作,方案成色试金石)

SSOT 的工程形态 = 知识单元(knowledge unit)。四步:

1. **找知识枢纽**:业务知识围绕什么对象旋转?零售 / 消费品 = SKU,专业服务 = 项目 / 案例,制造 = 机型 / 工艺,售后 = 故障码 / 工单类型。枢纽通常在诊断的资产盘点"冲突源"一列自己冒出来——冲突最密集的对象就是枢纽。
2. **定义单元 Schema**:1 枢纽实例 = 1 单元 = 唯一权威版本;字段化拆成 稳定事实(基础资料)/ 受控表述(合规审定话术,按渠道变体)/ 关联知识(FAQ、使用方法、关联推荐)。
3. **挂治理属性**:每单元 1 owner、版本与生效期、密级、复审周期——代谢与权限长在单元上,不是长在文件夹上。
4. **定消费映射**:门户 / 培训课件 / 详情页 / AI 语料全部从单元取数——"全渠道引用同源"是答案一致性的工程保证。

> 试金石:L2 一节如果只有"建立 SSOT 机制、制定元数据规范"这类动词短语,没有知识单元的具体 Schema,那还是 PPT,不是方案。

### 主数据边界纪律(必须明示,否则项目膨胀)

知识单元覆盖业务对象(SKU / 客户 / 物料)后,会与既有 PIM/ERP/CRM 产生权威源边界。方案必须写明:**知识库管"知识与表述"的权威源;交易主数据(价格 / 库存 / 编码)权威源留在原系统,知识单元单向引用、不反写。** 不写明,项目会悄悄膨胀成主数据治理项目(另一个量级)。

## 2. 用什么建:选型决策树(四层过滤)

顺序不可颠倒。前三层是组织约束,功能比较放最后——先用约束把候选砍到 2-4 个,再比功能。

1. **生态锚定**:员工每天驻留哪个协同生态(企微 / 钉钉 / 飞书 / Teams / 无)?消费入口必须长在员工已驻留处。跨生态迁移是另一量级的变革项目,除非客户主动提且 O5≥3,不夹带。
2. **数据敏感度与合规形态**:无硬约束 → SaaS 全集;部分敏感(客户数据 / 配方工艺)→ SaaS + 敏感域隔离或混合;硬性私有化 → 私有化产品集。上私有化同时查 D5——IT 独木型上私有化是"处方与体质冲突",必须写明增编前提。
3. **组织承接力匹配**:IT ≤3 人无数据岗 → 纯 SaaS + 厂商托管;IT 5-15 人有 1-2 数据岗 → SaaS 为主 + 轻量自建管道;有独立数据团队 → 全谱系可选。
4. **产品功能比较**:候选剩 2-4 个,再比结构化能力 / 检索质量 / API 开放度 / AI 成熟度 / 价格。

### 输出格式纪律

**永远给 2-3 个选项 + trade-off 矩阵,标注推荐项及理由,但把决定权留客户。** 替客户做唯一决定 = 替客户承担选型失败的全责。矩阵维度建议:构成 / 生态集成 / 结构化检索 AI / 运维要求 vs 承接力 / 成本(首年 + 三年)/ 主要风险。推荐项配三条理由 + 一句"如果〔条件变化〕则改选〔备选〕"。

## 3. 按什么顺序建:分期路线(四期)

标准四期,退出标准没达成不开下一期。每期结构固定:目标 → 范围(做什么 / 明确不做什么)→ 交付物验收口径 → 退出标准(量化闸门)。

### Phase 0 · 速赢(60-90 天)

- **存在意义 = 政治燃料**。在赞助人热情半衰期(约 6 个月)内给出看得见、讲得出的成果,把口头支持变成 Phase 1 的预算与编制。这是"老板热情型"病灶的解药。
- **场景选择标准**:可见度 × 确定性 × 单点闭环(不依赖跨部门改造)。从诊断的受益人测试答案里挑。
- **范围纪律**:单场景、单部门、白名单语料;明确写"不做什么"。
- **设计要求**:即使后续全停摆,Phase 0 产物本身要有独立残值。
- **退出标准示例**:场景上线且周活 ≥ 目标群体 X%;赞助人完成一次对内 / 对外展示;Phase 1 预算与运营编制获书面承诺。

### Phase 1 · 地基(3-6 月)

- L2 治理层主体(SSOT / 元数据 / 审核流 / 代谢 / 运营角色到岗)+ 高价值知识域清洗结构化(按"高频 × 高价值"矩阵排序,**不做全量迁移**)。
- **退出标准示例**:核心域 SSOT 覆盖 ≥80%;采样过时率 ≤10%;审核流跑通且有度量;运营角色到岗满 2 月。

### Phase 2 · 扩面(3-6 月)

- 知识域横向扩展 + 消费入口铺开(IM 内嵌 / 门户 / 培训挂接)+ 度量体系上线。
- **退出标准示例**:周活 / 搜索量达标;"群里问"对照下降;第二批知识域过质检线。

### Phase 3 · AI 深化(持续)

- 白名单语料上扩 AI 消费面(对内 Copilot → 对外客服 / 导购)+ 反馈闭环(badcase → 语料修正)+ 护栏与人工抽检。
- **前提**:A1 语料质量过线 + G1 审核流稳定。两前提不满足时,Phase 3 内容出现在任何更早阶段都是事故(除非 Phase 0 限定语料的内测形态)。

#### 对外 AI 监管前提(中国境内不可绕过)

向境内公众提供生成内容,需履行**生成式 AI 备案**(《生成式人工智能服务管理暂行办法》);算法推荐 / 深度合成另有算法备案。对内员工工具一般豁免,对外(会员 / 客户)适用。两条路径:

- **路径 A**:接入已备案大模型 API 并以其名义提供(多数中型企业的现实选择,合同确认备案覆盖范围)。
- **路径 B**:自建并自行备案(周期数月,需法务 + 算法团队)。

语料含个人信息(客服对话 / 健康肤质咨询)时,需个保法合规设计(去标识化 / 单独同意 / 匿名化)。方案里补一个动作项:"面客前确认备案路径"。

### 政治兼容性设计(分期层面)

真问题与陈述需求是"症状 / 错位表达"关系时,路线要给陈述需求一个体面位置。客户要 AI、诊断说地基烂——不要写"先做两年治理再上 AI"(等于让赞助人空手两年),而把 AI 以可控形态(限定语料内测)前置进 Phase 0,让赞助人始终"有 AI 可讲",工程重心仍压在 L2。

> **方案的叙事顺序可妥协,工程的依赖顺序不能妥协。**

## 4. 谁来养:运营机制

回答一个具体问题:上线后第 181 天,谁在干什么。写不出这个,方案就是"隐形运营"反模式。

### 最小完整集角色表(中型企业基准)

| 角色 | 编制形态 | 职责 |
|---|---|---|
| 知识运营官 | 1 专职(Phase 1 起硬性) | 内容质量、代谢执行、度量看板、需求收集 |
| 域知识管理员 | 每核心域 1 兼职(业务部门内,10-20% 工时,**写入岗位职责**) | 本域内容生产组织与初审 |
| 合规审查人 | 复用现有合规 / 法务岗,嵌入审核流 | 强监管内容发布前审查 |
| 平台管理员 | IT 兼职 | 账号、权限、集成技术维护 |

### 配套机制

- **贡献激励**:与绩效 / 晋升挂钩的轻量积分;对"知识 = 个人壁垒"者(销冠 / 老师傅)设计利益补偿式激励(署名传承 + 专项奖金 + 评优加分),把绝活显性化换成体面交易,而不是硬拔。
- **季度知识体检**:采样质检常态化。
- **badcase 通道**:一线一键报错 → 语料修正闭环。

### 预算纪律

**运营人力成本必须显性出现在预算表,且按 ≥3 年展示。** 隐去运营人力的方案是账面骗局——平台买得起,养不起,第二年就荒。
