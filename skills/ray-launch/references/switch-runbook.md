# references/switch-runbook.md —— 域名/DNS 切换 Runbook

整个上线里最需要**按序执行 + 每步验证 + 备好回滚**的一段。适用:要把域名从一个托管切到另一个(FOHO:Vercel → CMS 主机 SiteGround),或首次把域名指向线上站。纯静态站若不涉及迁域名可跳过本文件。

铁律先行:**任一步异常 → 立刻回滚(DNS 改回 + 平台重绑),不硬着头皮往下走。** 清理动作全部后置到观察期结束,就是为了保住这条回滚路。

---

## T-2 天:预备(切换前两天)

- **DNS TTL 调低到 300s**:让切换和回滚都能 5 分钟内全球生效。这是回滚能快的前提,必须提前改(TTL 本身也要等旧值过期才生效)。
- **内容后台做一次基线备份**:手动全站备份,出事有干净还原点。
- **性能调优**(内容冻结后再做,避免边调边改互相干扰):
  - 开缓存/加速(动态缓存 + 浏览器缓存)
  - **先别开 CSS/JS 合并压缩**——和可视化编辑器兼容坑多,要逐项开、逐项回归,别一次全开
  - 字体本地托管(别外链 Google Fonts)、图片批量压缩
- **冻结线上旧版**(不再改动),和客户约切换时间窗:选**目标客户的低峰时段**(欧美买家 = 北京下午)。

## T-Day:切换(严格按序,每步验证过了再走下一步)

### 1. 后台域名替换(切 DNS 之前先把库里的旧域名换掉)

- 后台常规设置:站点地址从 `cms.<域名>` 改成 `<域名>`
- **全库 URL 替换**(Better Search Replace 这类工具):先 dry-run 看命中计数(含 postmeta/options),确认无误再执行
- 固定链接重存一次,刷新 rewrite 规则
- **可视化编辑器的数据里单独存了 URL**:必须单独跑编辑器自己的 Replace URL + 重生成 CSS & Data。漏了这步,页面里还是旧域名。

### 2. DNS 切换

- `<域名>` 的 A 记录:旧 IP(Vercel)→ 新 IP(CMS 主机);`www` 的 CNAME 同步
- **⚠️ 邮箱记录一个都不能动**:MX / SPF / DKIM 保持原样(FOHO 邮箱在网易企业邮,不在主机商)。动了 = 客户收不到询盘,比站挂更严重。
- DNS 生效后,在新主机签发 apex + www 的 SSL 证书(Let's Encrypt)

### 3. 旧平台退役(防双站)

- 从旧平台(Vercel)**移除域名绑定**,避免同一域名两处响应造成解析漂移
- 项目本身保留、停部署即可,别删——回滚还要用

### 4. 上线后即时验证(切完立刻逐项过,别信控制台绿灯)

- `https://<域名>` 首页返回 **200** + 全板块在
- **View Source 搜旧域名(`cms.` 子域)= 0**——有残留说明库里 URL 没换干净,线上还在偷偷请求旧后台
- 每个关键页 200:`/products`、`/products/<slug>`、`/about`、`/contact`、`/faq`
- **表单提交 → 目标收件箱真的收到一封**(SMTP/发信已配为前提)。这是 B2B 站命根子。
- `sitemap.xml` 正常且 URL 为新域名 → **Google Search Console 重新提交**
- 旧子域(`cms.<域名>`)**301 到** `<域名>`(媒体旧链兜底)
- SSL 无混合内容告警(浏览器控制台看)
- 移动端抽查 2 页

> 验证取资源一律带随机 query(`?v=$(date +%s)`)绕 CDN/静态缓存,否则拉到旧文件会误判"没生效"。

## 回滚预案(任一步炸了)

1. DNS 改回旧 IP(Vercel),TTL 300s → 约 5 分钟全球生效
2. 旧平台(Vercel)重新绑定域名
3. 站即恢复到切换前状态

**正因为要保住这条回滚路,所有清理动作(删插件、退役字段、撤密钥、删 GraphQL 端点)必须放到观察期结束后**,切换当天一律不碰。

## T+1 ~ T+14:观察期

- 盯 **Google Search Console** 覆盖率:收录进展、404 报告
- 看主机访问日志的高频 404 路径,针对性补 301
- 稳定后再收尾清理:
  - 停用 / 删除闲置插件
  - 退役 headless 期的取数端点(GraphQL)、退役临时字段组——但**全局联系方式这类还在用的字段组保留**
  - **撤销上线期临时开的应用密码 / 卸载临时集成工具**(安全交接项,别把施工期的后门留在客户站上)
- DNS TTL 调回正常值(如 3600+),降解析压力
