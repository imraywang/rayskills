# X Articles 浏览器能力契约

## 选择顺序

1. 当前宿主已经提供、且能复用用户登录态的浏览器控制 Skill 或工具。
2. 当前宿主提供的电脑控制能力，可以在用户可见的浏览器中点击、输入、上传文件和读取页面状态。
3. 都没有时，在进入编辑器前停止，说明缺少什么能力；不得用网页抓取、普通 X API 或新建无登录浏览器冒充完成。

宿主可能是 Claude Code、Codex、WorkBuddy 或其他兼容 Agent。不要在本 Skill 中写死某个宿主的工具名称；加载宿主对应的浏览器或电脑控制说明并遵守其截图、定位和用户接管规则。

## 最低能力

所选能力必须同时做到：

- 打开并保持 `x.com` 的现有登录状态。
- 读取页面可见文字和当前 URL。
- 通过可见控件点击 `Articles`、`Drafts`、`Write`、`Preview` 等入口。
- 向标题和富文本正文输入内容。
- 通过文件选择控件上传本地 5:2 封面。
- 等待并确认保存状态。

缺少任意一项都不能承诺完整写入。若只能准备正文和封面，交付本地包并说明浏览器步骤尚未执行。

## 操作纪律

- 每次交互前读取当前页面；界面变化后重新观察，不复用旧坐标。
- 优先按可见文字、角色和标签定位；坐标只用于同一张最新截图中的临时点击。
- 不通过注入脚本绕过文件权限、登录、发布确认或编辑器限制。
- 发布按钮始终保持未点击，除非当前任务另有明确授权；本 Skill 默认只保存草稿。
