# 方案蓝图模板 + 示范样本

交付文档的骨架(8 节)+ 一份真实实战方案(青屿·商品知识中台)的落地样本,用来对照"长什么样才叫写透了"。执行时:先按 8 节结构起草,再拿每节样本校准颗粒度。

## 蓝图结构(8 节)

```
0. 方案摘要(1 页)      一句话方案 + 总路线图一页图 + 首期承诺(90 天交付什么/怎么验收)
1. 设计前提(半页)      约束三角写明 + 继承诊断结论(可行性评级与变绿条件如何被处理)
2. 目标架构(2 页)      四层具象化 + 知识单元定义 + 主数据边界声明 + 架构图(标注每组件哪期建成)
3. 选型建议(2-3 页)    决策树走查 + 候选对比矩阵 + 推荐与三条理由 + "如果〔条件变化〕则改选"
4. 分期实施计划(3-4 页) 每期固定结构;Phase 0 含选场景理由 + 范围 + 验收口径 + 退出标准(标责任方)
5. 运营机制(1-2 页)    角色编制表 + 激励考核 + 度量看板 + 第 181 天的一天
6. 预算与资源(1 页)    矩阵(项目 × Phase × 年运营),运营人力显性、按 3 年 TCO、数字标假设
7. 风险与前提(1 页)    每条 风险→触发信号→缓解动作→责任方;继承诊断 gating 项放最前
8. 下一步(半页)        三个待拍板决定 + 启动后两周动作清单 + 双方第一个里程碑会议时间
```

写作要点:
- **第 0 节的总路线图**是全案的锚。一页看懂:时间轴 × 各期目标 × 预算量级 × 退出闸门。
- **第 4 节退出标准必须标责任方**(甲方可控 / 乙方可控 / 双方共担)。依赖客户侧行为的指标,必须挂前提条件,否则会被读成乙方的交付承诺——闸门是双方里程碑,不是乙方军令状。
- **第 5 节"第 181 天的一天"**:写一段上线半年后运营者的日常,让客户具象看到"养"是什么。这一段比任何角色表都有说服力。
- **第 6 节所有数字标假设**,给量级不给精确报价(精确报价留到范围锁定后)。

## 示范样本(青屿·商品知识中台)

真实账本:一家化妆品 / 零售企业,要"AI 客服",诊断查出地基烂(话术多版本冲突、无合规闭环、无运营机制),红灯。方案没有顺着做"全面 AI 建设",而是把 AI 收进限定语料内测、工程重心压在商品知识中台。以下是各节落地样本。

### 一句话方案

> "以『商品知识中台』为唯一可信源、以合规审定为闸门、以企微为消费入口,四期建成人与 AI 共用的知识体系——第一期 90 天,在山茶系列单品类上跑通全链路并交付限定语料的 AI 内测。"

### 总路线图(ASCII 范例)

```
时间轴 ──90天────┬─至+9月─┬─至+15月─┬─持续─
      Phase 0速赢 │Phase 1地基│Phase 2扩面│Phase 3 AI深化
      山茶系列试点 │全品类SSOT │SOP/培训/私域│Copilot全量
      25-45万     │60-120万   │45-80万    │按场景立项
退出闸门 试点5项验收→编制到岗+审核流稳→度量达标→(语料质检线+护栏)
```

### 知识单元实例——"商品卡"

商品卡 = 基础资料(自 PIM)+ 成分与功效『合规审定版』+ 卖点话术(按渠道变体:门店 / 直播 / 私域 / 客服)+ FAQ + 使用方法与禁忌 + 关联推荐。每张卡 1 owner、1 生效版本、全渠道引用同源。署名机制(生产者署名可见)回应话术生产方"专业产出被收编"的顾虑——统一的是口径,不是抹掉出处。

### Phase 0 退出标准表(含责任拆分,示范"验收口径而非单方军令状")

| # | 指标 | 目标 | 责任 |
|---|---|---|---|
| 1 | 商品卡覆盖与审定 | 100%(法务终审可查) | 乙方主导(前提:法务审定 ≤3 工作日) |
| 2 | 客服答案一致性(复测扩样 40 题) | ≥90%,违规 0 | 乙方主导(前提:客服按新 FAQ 培训到位) |
| 3 | 新品知识包到店时效 | ≤3 天 | 双方共担(前提:市场部按流程提供物料) |
| 4 | BA 内测群周活 | ≥60%,AI 准确率 ≥90% | 双方共担(内测组织激励由零售 VP 牵头) |
| 5 | 政治成果 | 林总 1 次展示 + Phase 1 预算 + 运营官编制书面承诺 | 甲方决策 |

> 责任拆分说明:闸门是双方里程碑,不是乙方军令状——任一"前提"失守,对应指标口径重新议定。

### 选型三选项矩阵(范例)

A 腾讯系组合〔推荐〕/ B 独立知识库 SaaS + RAG / C 私有化开源。维度:构成、企微集成、运维要求、合规可控、成本(首年)、主要风险。推荐 A 三理由 + 改选条件(12 月内组数据团队 ≥3 人且对外 AI 体量起来,则 Phase 3 重评 C)+ B 作 fallback。

### 预算表(范例,内部人力单列并标注按到岗折算)

行:平台工具 SaaS 年费 / 实施与咨询服务 / AI 资源(内测→扩量)/ 内部人力(显性化)/ 合计。列:Phase 0 / Phase 1 / Phase 2 / 年运营稳态。

- Phase 0 = 25-45 万,Phase 1 = 60-120 万,Phase 2 = 45-80 万,年运营稳态 = 50-85 万 / 年。
- 三年总拥有成本量级:220-390 万(含人力),对照失败成本(BA 培养周期、新品落地提前、合规事故代价)做投资回收测算。

### 运营激励设计(范例)

复制该企业已成功的"带教费"模式:商品卡贡献与采纳计积分,季度兑换 + 纳入晋阶答辩。**销冠话术工作坊**以"署名传承 + 专项奖金 + 评优加分"交换绝活显性化——回应"写出来人人都会我算什么":写出来,署你的名,算你的传承。

### 第 181 天的一天(范例)

知识运营官晨会看板五分钟(昨日搜索 Top10 未命中 → 2 个建卡任务派域管理员);处理 3 条 BA 报错(1 条价格过期 → 触发卡更新 → 全渠道同步);下午新品上市会 → 知识包 T+1 发布排期;周五给法务送审 6 张卡的功效话术变体。

## 反模式清单(交付前自检,逐项打勾)

| 反模式 | 纠正动作 |
|---|---|
| 大平台开场 | 机制先于平台 |
| 全量迁移 | 用过时率数据砍范围 |
| 先验分类学(花两月设计完美分类体系) | 演进式生长 |
| AI 直饮生水(RAG 直连未治理源) | 加白名单语料机制 |
| 隐形运营(预算无运营人力) | 补上,按 3 年展示 |
| 超体质处方(运维要求超 D5) | 降形态或写明增编前提 |
| 唯一答案(选型只给一个) | 补 trade-off 矩阵 |
| 无退出标准 | 每期补量化闸门 |
| 生态夹带(借项目夹带 IM / 协同迁移) | 拆出去单独评估 |
| 无备案面客 | 补生成式 AI 备案路径确认动作 |
| 集中即裸奔(集中后无密级分层与导出控制) | 补 G3 侧设计 |
| 第 181 天无人 | 回运营节重做 |

青屿版逐项 checklist(可直接套):非大平台开场 / 无全量迁移 / AI 不直饮生水 / 无备案不面客 / 集中不裸奔 / 运营成本显性 / 不超组织体质 / 选型有 trade-off / 每期有退出标准且责任拆分 / 第 181 天有答案——逐项打勾,缺一项方案不交付。
