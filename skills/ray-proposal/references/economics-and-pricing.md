# 单位经济 · 预算量级 · 报价结构

预算这件事,方案里给的永远是"量级 + 假设",精确报价留到范围锁定之后。下表是校准用的锚,不是报价单——照抄进客户方案是懒,要按这家企业的实际范围推。

|---|---|---|
| 评估诊断 | ¥8-25 万 | 视规模深度 |
| Phase 0 速赢(含实施) | ¥20-60 万 | 含平台首年费 + 场景实施 |

## 内部人力折算纪律

内部人力必须显性列进预算,且按各阶段**实际到岗月数**折算(例:运营官 Phase 1 中段到岗,则计半年)。表注写明折算口径。

- 整年挂账 = 虚高,让 TCO 假大。
- 隐去不计 = 自欺,平台买得起养不起,第二年荒。

两种都错。真实折算 + 口径透明,才是能过财务那关的账。

## 单位经济 / 投资回收

方案要能对着失败成本算一笔账,而不是只报支出。

- **支出侧**:三年总拥有成本(TCO)= 平台年费 + 实施咨询 + AI 资源 + 内部人力(按到岗折算),含运营、按 3 年展示。
- **收益 / 避损侧**:对照诊断里挖出的失败成本——培养周期(如 BA / 新人成长慢)、落地时效(新品知识到店从 3 周压到 3 天)、合规事故代价(一次功效宣称客诉的赔付 + 商誉)。
- **回收测算**:用可避损的量级 × 发生频率,对照 TCO,给一个"多久把这笔钱赚回来"的区间。给区间不给假精确。

## 报价结构纪律

1. **量级 + 假设,不是精确数字**。每个数字后挂它成立的前提("按 3 个核心域、200 个账号计")。范围一变,数字重算——这样报,后面加钱有据,不背锅。
2. **分期报价,不是一口价**。Phase 0 单独可立项、可验收、有独立残值——让客户用小钱验证再决定追加,这本身是对客户负责,也是把一锤子买卖变成长期关系的结构。
3. **咨询与工具分列**。诊断 / 方案 / 实施陪跑(咨询收入)与平台 SaaS / AI 资源(工具成本)分开列,客户看得清哪部分是买你的判断、哪部分是买厂商的东西。
4. **运营人力永远在表内**。这是把"能不能长期活"提前摆到台面的地方,不是可选项。
