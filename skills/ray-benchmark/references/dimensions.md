# 五维解剖 · 拆解 checklist

执行步骤二时的查表依据。每一维给三样东西:**该看什么**(可观察清单)、**去哪找证据**(优先可验证源)、**可迁移性拷问**(这一维专属的、决定"能不能抄"的关键一问)。

一条纪律先立在最前:**填的是事实,不是形容词。** "定价很聪明"不是拆解,"入门 $X/月含 N 次调用、超出按 $Y/次、企业版隐藏定价需联系销售"才是。形容词是你偷懒的信号。

## 一、产品形态

它到底卖的是什么、边界画在哪。

**该看什么**
- 交付形态:SaaS / API / 插件 / 工具 / 内容订阅 / 社区 / 服务,还是组合
- 核心功能:一句话能说清它替用户完成的那件事是什么(说不清就是它自己也没想清,或你没拆透)
- 边界取舍:它**刻意砍掉**了什么?砍掉的往往比做了的更能说明定位
- 首个 "aha 时刻":用户第一次感到"值"是在哪一步、要走多久到那步
- 目标用户画像:谁在付钱,和谁在用,是不是同一批人

**去哪找证据**
- 官网首页 + 功能页 + 更新日志(changelog 暴露它把资源押在哪)
- 自己走一遍注册到首次使用的完整流程,记下卡点和惊喜点
- 评价区 / 社区 / 差评:用户实际拿它干什么,常和官方定位不一样

**可迁移性拷问**
> 它的产品形态,是任何人想清楚都能搭出来的(🟢 机制),还是依赖它独有的技术 / 数据 / 资质才成立(🔴 禀赋)?"套壳一层就能复刻的形态" ≠ "背后那份数据 / 分发也能复刻"。

## 二、定价与变现

拆定价是拆商业模式最快的入口——价格结构里写着它认为自己值什么、赌哪种用户掏钱。

**该看什么**
- 定价结构:免费 / 订阅 / 按量 / 一次性 / 分层,各层的锚点和跳档设计
- 价格锚:它用什么让中间那档显得"最划算"(经典三档诱饵)
- 免费→付费的转化钩:免费版卡在哪、用什么逼你升级(额度 / 功能 / 协作 / 品牌水印)
- 真正赚钱的那条线:表面卖 A,利润可能来自 B(企业版 / 增值服务 / 抽成 / 数据)。找到主现金流
- 隐藏定价:企业版"联系销售"= 高客单,量级信息藏在这

**去哪找证据**
- 定价页(可验证)、年付 vs 月付折扣、历史定价变化(archive 快照能看涨价轨迹)
- 财报 / 融资新闻里的营收结构(可验证);"月入 X"类自述(未验证,打折)

**可迁移性拷问**
> 它的定价能成立,是因为**定价结构本身设计得好**(🟢,可照搬结构),还是因为它有你没有的**议价权 / 品牌溢价 / 存量用户基数**(🔴,同样定价你收不上来)?先抄结构,别抄它的价位。

## 三、增长引擎

大多数成功对象只有**一台主引擎**,其余都是装饰。找错引擎 = 抄错方向。

**该看什么**
- 主引擎判定:内容 / PLG 产品自传播 / 付费投放 / SEO / 社区 / 分销联盟 / 生态搭车——哪一台带来了大部分增长
- 增长循环:能不能画出一个自我放大的环(用户→产出→带来新用户)。有环 vs 靠持续输血,天差地别
- 冷启动手段:它最初 0→1 那批种子用户从哪来(常是一次性、不可复制的,注意别当常规打法)
- 增长的时间线:是线性爬升,还是某个时点陡增(陡增点背后常有一次性红利 / 事件,是 🔴 信号)

**去哪找证据**
- 流量来源估算(SimilarWeb 类,估值,标未验证)、外链结构、社区活跃度
- 官方复盘 / 创始人访谈里的"我们怎么涨起来的"(注意:回忆常美化,幸存者视角)

**可迁移性拷问**
> 这台引擎依赖的燃料你有吗?"靠创始人 10 万存量粉丝冷启动" 的引擎,你没那 10 万粉就点不着(🔴)。把**引擎结构**(🟢,可借)和**它的启动燃料**(常 🔴)分开看——结构能学,燃料要自己找。

## 四、内容与渠道打法

尤其对内容 / IP 线对标对象。拆的是"可复制的公式" vs "不可复制的个人魅力"。

**该看什么**
- 发什么:选题母题、内容类型配比、有没有可识别的模板 / 公式
- 在哪发:主阵地 + 分发矩阵,哪个平台是主场
- 什么节奏:频率、时段、连载 vs 单发
- 钩子:开头怎么抓人、标题公式、封面 / 缩略图套路
- 人设 vs 方法:哪些效果来自可拆的结构(🟢),哪些来自这个人独有的经历 / 声音 / 长相 / 口条(🔴)

**去哪找证据**
- 翻它最近 20-50 条内容,标出**爆款 vs 平庸**的差异(样本要够,单条爆款可能是运气)
- 看爆款的评论区:观众到底为什么转发 / 收藏(转发理由 = 可抄的价值点)
- 别只看它想让你看的精选,看它的日常均值

**可迁移性拷问**
> 把这个人换成你,同样的选题 + 结构 + 钩子,还成立吗?成立的部分是公式(🟢),不成立的部分是个人魅力(🔴)。**审美纪律**:你不喜欢它的风格,不代表它的公式无效——把"我不喜欢"和"它不管用"分开。

## 五、护城河

最容易被吹嘘、也最需要冷眼的一维。很多"护城河"其实只是"暂时领先"。

**该看什么**
- 护城河类型排查:网络效应 / 数据飞轮 / 规模成本 / 品牌心智 / 转换成本 / 独家资源 / 技术代差
- 真伪测试:如果一个资金充足的团队今天下场,多久能追平?追不平的地方才是真护城河
- 领先 vs 壁垒:它现在赢,是因为别人抄不动(壁垒),还是只是先到(领先,会被抹平)?
- 禀赋型护城河点名:牌照 / 独家数据 / 独家渠道 / 先发时机——这些是最强护城河,也是对你最 🔴 的东西

**去哪找证据**
- 有没有后来者已经追平部分功能(追平了 = 那部分不是护城河)
- 它自己在防守什么(法务 / 封锁 / 绑定合同,暴露它认为哪里最怕被抄)

**可迁移性拷问**
> 它的护城河,恰恰是你**最抄不走**的部分(否则就不叫护城河了)。所以这一维的产出不是"抄护城河",而是**回答两问**:①它的护城河是真的还是暂时领先?②如果是真的,你要找的是**另一条**它没占的护城河,而不是硬撞它的墙。护城河维度指导的是"绕道",不是"照抄"。

## 数据可信度分级(公用)

拆解每一维时,给引用的每个数字标一个可信度等级,直接写进报告:

| 级 | 来源 | 处置 |
|---|---|---|
| **可验证** | 官方定价页、财报 / 融资公告、平台显示的公开粉丝 / 播放数、第三方审计 | 可作行动依据 |
| **估算** | 第三方流量估算工具、行业报告推算、公开信息反推 | 标"估算",作参考不作依据 |
| **未验证** | 自我宣称的营收 / 转化率 / 增速、访谈里的回忆数字、无旁证的截图 | 默认打折,明标"未验证",**不作行动依据** |

数字之间对不上时,先怀疑数字来源,再考虑解释。给幻觉编一个合理的故事,是对标拆解里最常见、也最贵的错误。
