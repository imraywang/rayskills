# x-draft — 每日声部 C 候选装配

产出 8-12 条以 Ray「温暖的同行者」风格写成的**声部 C（社会观察 / 人生哲理）**完整稿候选，落盘队列文件并在会话里展示。Ray 只做终审：挑 1-3 条、改血肉、手动复制粘贴发布。

> **2026-07-10 转向**：本管线只产声部 C。声部 A（科技实测）/ B（出海搞钱）依赖 Ray 的真实操作与数字，机器保证不了质量，由 Ray 亲自出，**不在本 skill 范围**。声部 C 是纯哲理 / 观察，**不需要真实依据**，可基于选题库 + 知识库直接生成完整稿——这正是它可机器化的原因。

## 固定资产（每次必读）

- 风格卡：`~/projects/content/x-pipeline/style/STYLE.md` —— 声部 C 单焦点、四母题、四体裁、硬指标、口吻、红线
- **选题库**：`~/projects/content/x-pipeline/style/topic-bank-C.md` —— 28 选题 × 4 母题 + 钩子模板 + 真实种子；本批优先从这里取
- Few-shot 库：`~/projects/content/x-pipeline/style/fewshot-bank.md` —— 取声部 C 同体裁样例
- Morris 对标：`~/projects/content/x-pipeline/data/morris/morris.jsonl` + `analyze_morris.py` —— 高赞角度地图（借形式不借内容）
- 历史推文：`~/projects/content/x-pipeline/data/tweets-history.jsonl` —— 去重依据 + Ray 真实立场
- 往期批次：`~/projects/content/x-pipeline/queue/*.md` —— 近 7 天批次必须去重

## 流程

### 1. 选题（从库取，不做热点采集）

- **主源**：`topic-bank-C.md`，四母题（长期主义·延迟满足 / 祛魅·平视 / 小胜利·稳定内核 / 利他·处世厚道）各取若干，覆盖均衡。
- **续扩**：库里选题用尽或想要新角度，参考 Morris 选题地图 + 模型知识库，按母题生成新选题（仍是纯哲理，无需由头）。
- **灵感参数**：用户在 /x-draft 后带的文字是当日观察，**最高优先级**，优先做成候选。
- **利他母题**历史写得最少（是选题空缺），每批刻意保证 ≥2 条。

### 2. 生成（8-12 条完整稿）

- 先读 few-shot / 选题种子再动笔。
- **四体裁轮换**：断言金句体（主力，两拍收）/ 第一性原理拆解体（Ray 独门，「X 的第一性原理是 A 不是 B」）/ 下定义递进体（「X 的本质是 Y」）/ 观察 → 洞察体。
- 长度两档：≤50 字短断言，或 150-280 字结构化中长；中长占比 ≥1/3。
- **声部 C 无需由头**——纯哲理 / 观察直接写，不必「刷到 / 有人发我」开题。仅当某选题挂 Ray 的真实经历（`topic-bank-C.md` 里标 `[需要你的真实案例]` 的），才留占位给 Ray，机器不编。
- 每条标注：母题 / 体裁 / 长度带。

**红线（违者废稿）**：

- **不虚构第一人称经历、数字、收入、成本**。纯哲理是抽象断言与普遍观察，**不假托 Ray 的具体经历**；「我实测 / 我赚了 Xw」属声部 A/B，声部 C 不出现。
- **温度红线**：温暖的同行者，平视、给底气；不居高、不刻薄。冷峻到批判是 Morris 不是 Ray。收在鼓励 / 可执行下一步，禁双金句叠加。
- 「不是…而是」全批每条 ≤1 次；无「宝子 / 家人们」淘宝腔；写 𝕏 不写 Twitter / 推特。
- 与历史推文 + 近 7 天批次查重：同一断言不重复出。

### 3. 质检

1. 候选写成 JSON `[{id, text, voice:"C-体裁"}]`，跑 `node ~/projects/content/x-pipeline/scripts/lint.js <file>`，issues 必须清零（warns 看情况）。
2. 模型自审两遍：① **声线匹配分**（像不像「温暖的同行者」Ray，1-10）② **AI 味分**（10 = 无 AI 味）。任一 <7 就重写或废弃。评审**无视 `[需要你的真实案例]` 占位符**。
3. 如当前会话可用 Workflow（ultracode），可 fan-out 独立评审 agent 做对抗性质检替代自审。

### 4. 交付

- 落盘 `~/projects/content/x-pipeline/queue/YYYY-MM-DD.md`，格式见下。
- 会话里按「今日推荐 Top 3 → 其余按母题分组」展示，每条给一行推荐理由。
- 结尾提醒：发布后表现突出 / 扑街的，人工记给 playbook（Phase 3 数据回流前）。


## 迭代纪律

Ray 打回的每条意见，判断是个案还是规律：规律 → 当场改 `STYLE.md` / `topic-bank-C.md` / 本文件，不要只改稿。Phase 1 目标：把风格卡校准到批次通过率 ≥ 1/3。
