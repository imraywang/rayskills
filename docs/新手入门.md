# rayskills 新手入门

rayskills 是 Ray 在真实业务里沉淀的 AI Skill 工具箱。不是"教你怎么想"的诊断器,是**直接把下一步干掉**的 builder 工具——每个 skill 背后都是一段真实发生的实践。

## 第一次怎么用

不用记 skill 名。直接把真实处境交给 `/ray`:

```text
/ray 我有个客户想上 AI 客服,不知道能不能做
/ray 上周把被入侵的服务器清干净了,想发出去
/ray 帮我把这个 Next 站上线到 Vercel
```

`/ray` 读当前对话的信息,判断此刻最该做的一步,分发到对的成员 skill。每次只决定当前一步,做完再 `/ray` 让它基于结果决定下一步。

## 熟了直接调具体 skill

```text
/ray-diagnose 一家 800 人化妆品公司想上 AI 客服,老板热情但没立项
/anvil 宠物陪伴 / 25-35岁独居女性 / 79元 / 一二线城市
/ray-thread ~/projects/xxx/RETRO.md
/ray-vpsinit root@1.2.3.4
/ray-benchmark 拆一下 OpenRouter 是怎么做起来的
```

## Skill 全目录

| Skill | 干什么 |
|---|---|
| `/ray` | 主入口路由,不知道用哪个就交给它 |
| `/ray-vpsinit` | 新 VPS 开荒:加固 + 代理栈 + 交接 |
| `/ray-nodecheck` | 节点/中转链健康巡检(只读) |
| `/ray-thread` | 实战经历 → build-in-public thread 骨架(不代笔) |
| `/ray-tweet` | 每日社会观察/哲理推文候选 |
| `/ray-metrics` | X 账号数据周报 |
| `/ray-benchmark` | 对标拆解(产品/定价/增长,判可迁移性) |
| `/ray-report` | magazine 风格长文/报告(HTML+PDF+公众号) |
| `/ray-diagnose` | 企业知识库/AI 前置诊断(六维+红黄绿) |
| `/ray-proposal` 🔒 | 诊断结论 → 方案蓝图 |
| `anvil` | 从批判框架锻造产品概念 |
| `/ray-launch` | 落地页/B2B 站上线全流程 |
| `/ray-weekly` 🔒 | 每周复盘 |
| `/ray-cleanup` 🔒 | 项目归档 + 磁盘瘦身 |

skill 之间怎么衔接 → 见 [skill-link-map.md](skill-link-map.md)。

## 设计原则

1. **有尸体或有账本** — 每个 skill 背后要么是真实事故复盘,要么是真实收入业务。
2. **一 skill 一现金流线** — relay 引流 / 企业咨询 / IP 增长,不做纯炫技。
3. **skill 即内容** — 每个成熟 skill 配一条实战 thread,发布即营销。
